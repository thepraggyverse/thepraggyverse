# Thepraggyverse Vision

Thepraggyverse is a app project. This file captures where the project could go over time.

Treat this as direction, not implementation permission. Vision-only ideas must become requirements, then a plan, then receive user confirmation before code is written.

## North Star

- Describe the long-term product promise in plain language.
- Name who it should serve best.
- Name the feeling or workflow quality the project should protect.

## Current Scope

- Define the smallest useful product slice for the next implementation pass.
- Name what is intentionally out of scope for now.
- Keep the current scope smaller than the full ambition.

## Roadmap Horizons

### Now

- The smallest loop worth proving.
- The core behavior that must work before adjacent features matter.

### Next

- Improvements that are likely soon, but should still pass through requirements and planning.
- Product or technical questions that need discovery before implementation.

### Later

- Larger ambitions, integrations, platform expansions, automation, or polish ideas.
- Ideas that are exciting but should not distract from the current slice.

## Not V1

- List ideas that are deliberately outside the first version.
- Name any integrations, infrastructure, automation, or advanced workflows that should wait.
- Keep this section honest so future agents do not quietly expand the build.

## Safety and Philosophy Rules

- Keep runtime/generated data local and private unless explicitly approved.
- Do not add production infrastructure, billing, auth, telemetry, deployment, or external integrations from vision alone.
- Preserve the smallest useful loop before expanding the system.
- Surface trade-offs before expanding scope.

## Deferred Ideas

- Capture bigger ideas here instead of letting them leak into the active plan.
- Add a short note for why each idea is deferred.

## Promotion Rules

- `VISION.md` says where this could go.
- `docs/brainstorms/` decides what the product means right now.
- `docs/plans/` decides how to build the next confirmed slice.
- Implementation happens only after the user confirms the plan.
